## Design Context

### Users
- **Who**: 드라이브를 사랑하는 한국 사용자들 (운전자 + 동승자)로 구성된 동아리 커뮤니티
- **Context**: 모바일 우선 PWA 환경, 이동 중이나 드라이브 전후에 접속
- **Job to be done**: 드라이브 이벤트 참여/관리, 회원 간 소통, 차량 정보 공유, 편의 도구(주유소/주차장/코스 추천) 활용

### Brand Personality
- **3 Words**: 테크 / 스마트 / 미래지향
- **Voice & Tone**: 기술적이면서도 친근한 톤. 불필요한 장식 없이 정보를 명확히 전달. 데이터 중심적이면서도 커뮤니티의 따뜻함을 잃지 않는 균형.
- **Emotional Goal**: 신뢰와 안정감 — 잘 관리되는 스마트 플랫폼이라는 느낌

### Aesthetic Direction
- **Visual Tone**: 네오 브루탈리스트 기반의 테크 감성. 0px border-radius, 오프셋 box-shadow, 굵은 border-left/top 강조선
- **Theme**: 다크 모드 기본 + 라이트 모드 지원 (CSS variables 기반 듀얼 테마)
- **Color Palette**:
  - Primary: `#7c6fff` (보라) / Light: `#a09aff` / Dark: `#5a54e0`
  - Accent: `#ff6b6b` (레드)
  - Semantic: Driver `#4ade80` (그린), Passenger `#60a5fa` (블루)
  - Background: `#0a0a0e` → `#111118` → `#1a1a24` (다크 3단계)
  - Text: `#f0f0f5` → `#9898b0` → `#5c5c78` (3단계)
- **Typography**: Inter + Apple SD Gothic Neo, font-weight 600~800 주로 사용, letter-spacing -.1px~2px
- **Anti-references**: 과도하게 둥글고 파스텔 톤인 귀여운 UI. 지나치게 복잡한 대시보드 스타일.

### Design Principles

1. **Smart Brutalism** — 네오 브루탈리스트의 대담함을 유지하되, 정보 전달의 명확성을 최우선으로 한다. 장식보다 구조.
2. **Data-Driven Trust** — 숫자와 상태를 명확히 보여주어 사용자가 플랫폼을 신뢰할 수 있도록 한다. stat-card, vote-progress 등 데이터 시각화 요소 활용.
3. **Mobile-First Responsive** — PWA로 모바일 사용이 주요 시나리오. 터치 타겟 충분히 확보, 스크롤 기반 탐색 최적화.
4. **Consistent Semantic Color** — 운전자(green), 동승자(blue), 위험/부재(red), 보류(yellow)의 시맨틱 컬러 시스템 일관 유지.
5. **Progressive Disclosure** — 정보를 단계적으로 공개. collapsed 설명, 모달 상세, 필터로 복잡도를 관리한다.

### Tech Stack
- Vanilla HTML/CSS/JS (no framework, no bundler)
- Firebase (Auth, Firestore, Functions)
- Cloudflare Pages (hosting)
- PWA (manifest.json, service worker)
- External: CropperJS, Google Analytics, Microsoft Clarity
